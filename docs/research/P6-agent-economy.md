# P6: Agent-to-Agent Commerce / Autonomous Agent Economy

**Source:** Perplexity research, March 2026
**Status:** DONE
**Decision impact:** Core thesis validation

## Key Findings

- **Google A2A:** Real and in production for enterprise workflow orchestration, but NOT open-market agent commerce
- **Research:** "A2A Protocol with Ledger-Anchored Identities and x402 Micropayments" — extends A2A with blockchain identity + payments (prototype level)
- **Fetch.ai:** Demonstrated AI-to-AI payments (Dec 2025) but limited pilots, not mass usage
- **Crypto agent projects:** Hundreds exist, billions in token value, but most are speculative
- **MCP:** Gives discovery + invocation substrate, but NOT payment clearing
- **Agent wallets:** Agents hold keys or use MPC wallets. No signatureless native agent primitive exists
- **Legal:** No jurisdiction has regulated "agentic commerce" — gray zone. Keep human principal clearly on hook
- **Academic:** Theory for agent marketplaces and negotiation is MATURE. Gap is engineering + regulation

## RASHK Implications

- Core thesis (agents discover, contract, execute, pay) is EMERGING but not proven at scale
- RASHK would be an early mover — high risk, high reward positioning
- Design for human-approved agent transactions initially, fully autonomous later
- MCP provides service discovery but RASHK needs its own payment/clearing layer
- Agent wallet design should use capability-bounded keys (not full wallet access)
- Legal gray zone means keeping human principal clearly responsible is essential for now
- The engineering gap (not theory gap) is where RASHK can differentiate
